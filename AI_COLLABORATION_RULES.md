# AI Collaboration Rules

Короткий рабочий документ для любой сессии ИИ по этому репозиторию.

## Источники правил

Главный источник инженерных законов:

- [ENGINEERING_LAWS.md](/f:/Новая%20папка/ENGINEERING_LAWS.md)

Карта текущей структуры проекта:

- [ARCHITECTURE.md](/f:/Новая%20папка/ARCHITECTURE.md)

Если между документами есть различие:

1. `ENGINEERING_LAWS.md` важнее.
2. `ARCHITECTURE.md` описывает фактическое состояние.
3. Этот файл не вводит новые правила, а только упрощает рабочий режим.

## Область действия

Эти правила относятся ко всему проекту:

- Telegram-бот;
- групповые заявки;
- FastAPI backend;
- React-админка;
- проектный калькулятор;
- расчёты, спецификации и сметы;
- хранилище, конфигурация и AI-интеграции.

## Базовый режим работы

- Сначала смотри на текущую структуру репозитория, потом предлагай изменения.
- Не смешивай интерфейс, расчёты, сценарии, БД и интеграции в одном месте.
- Хендлеры, API-маршруты и UI-обработчики должны быть тонкими.
- Новую бизнес-логику не прячь в JSX, хендлерах и случайных `if`.
- Не усиливай монолитность `src/supply_bot/storage.py`, `src/supply_bot/services/dialogue.py`, `admin-ui/src/App.tsx` и `admin-ui/src/calculator.tsx`.
- Для нового Python-кода используй типизацию.
- Для нового TypeScript-кода не оставляй неявные `any`.

## Обязательный порядок в рабочей сессии

Перед реализацией:

1. Укажи, какие файлы будут изменены.
2. Укажи, какие файлы будут созданы.
3. Укажи, в каком слое проекта находится изменение.
4. Кратко объясни, зачем это нужно.

После реализации:

1. Объясни, что изменено.
2. Объясни, зачем это сделано.
3. Укажи реальные риски, хвосты или ограничения.

## Запреты по умолчанию

Без явного запроса не нужно:

- создавать служебные файлы “на будущее”;
- плодить новые абстракции без текущей пользы;
- дублировать существующую логику в новом месте;
- устраивать скрытый большой рефакторинг под видом маленькой правки.

## Короткий фильтр решения

Перед изменением проверь:

1. Это делает проект понятнее?
2. Это уменьшает связность?
3. Это будет поддерживаемо через несколько месяцев?
4. Это работает для всей системы, а не только для одной точки входа?
5. Это не создаёт новый god-file?

Если ответ слабый, решение нужно переделать.
