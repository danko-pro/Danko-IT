# Dashboard Scene Plan

## Текущая схема экранов

`dashboard` работает как `scene-layer` с четырьмя режимами:

- `overview` — карточка объекта
- `passport` — паспорт объекта
- `finance` — финансы
- `accounting` — таблица учета

## Границы экранов

### `overview`

Назначение:
- обзор объекта;
- мини финансовая сводка;
- быстрые сигналы по договору.

Состав:
- верхний `scene-switch`;
- signal-иконки;
- компактная мини-сводка.

Важно:
- мини-сводка остается только здесь;
- сюда не переносится форма паспорта объекта.

### `passport`

Назначение:
- ручной ввод исходных данных объекта;
- хранение параметров расчета.

Состав:
- название;
- адрес;
- квартира;
- этаж;
- лифт;
- въезд на территорию;
- код домофона;
- ответственный;
- площадь квартиры по полу;
- плановая маржа.

Важно:
- финансовые summary-блоки сюда не переносятся.

### `finance`

Назначение:
- операционная финансовая работа по объекту.

Состав:
- статьи;
- авансы;
- договор.

### `accounting`

Назначение:
- детальный учет объекта.

Состав:
- summary-strip;
- ledger table;
- документы и статусы учета.

## Текущий статус реализации

Уже сделано:
- общий `scene-switch`;
- отдельные сцены `overview/passport/finance/accounting`;
- сохранение паспорта объекта через backend;
- верхний `scene-chrome` живет внутри самих сцен, а не отдельной полосой над ними;
- выровнен верхний ритм сцен, чтобы вкладки не скакали при переключении.
