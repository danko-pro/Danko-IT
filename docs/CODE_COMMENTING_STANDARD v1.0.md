# Code Commenting Standard v1.0

## Главное правило

Комментарии и docstring в коде проекта пишутся на русском языке.

## Что комментируем

- назначение модуля;
- назначение класса;
- назначение публичной функции или метода;
- границы слоев и причины передачи управления в другой слой;
- неочевидные ветвления, fallback, проверки безопасности, ограничения размера;
- места, где AI, OCR, Telegram, storage или HTTP-route могут быть неправильно поняты при следующей доработке.

## Что не комментируем

- очевидные присваивания;
- каждую строку подряд;
- синтаксис языка;
- временные объяснения в стиле "здесь вызываем функцию";
- комментарии, которые повторяют имя переменной или функции.

## Формат

Для модулей, классов и публичных функций используем docstring.

Для сложных внутренних участков используем короткий inline-комментарий перед блоком.

Если код уже самодокументируемый, комментарий объясняет не "что делает строка", а "почему это важно для слоя, безопасности или сценария".

## Stop Line

Новый backend-код не считается законченным, если:

- в нем есть новые комментарии на английском;
- публичные классы и функции без docstring;
- сложные fallback/AI/storage/security ветки не объяснены;
- комментарии создают шум вместо пользы.
